=>Pipe ( | )
    ls                  //   -list
    ls -l               //   - vertical display (-l = long listing, display lots of information)
    ls -lh              //   - h = human formatted (diplay file size in a human-readable format)
    ls -lh | head       //   - Pipe (|)
    ls -lh | head -3    //   - first limited lines including -total filesize- line
    ls -lh | tail       //   - last 10 lines in the lists
    ls -lh | tail -2    //   - last limited lines in the lists
    
    ls -lF              (-F = Distinguish directories and files in linux)
    ls -lS              (-S = sort file by file size, biggest to smallest size)
    ls -l               (list order by name, a to z )
    ls -lt              (order by time, z to a )
    ls -l --full-time   (show by full time )

    ---------------------------------------------------------------------

        ls [options] [file...]
        
        -l:
        Common Options
        Description: Displays detailed information about each file and directory in a long format, including permissions, number of links, owner, group, size, and modification time.
        
        -h or --human-readable:
        Description: Displays file sizes in a human-readable format (e.g., K, M, G).

        _____________________________________________________________________
        command1 | command2
        
        command1: The command that produces output.
        command2: The command that takes the output of command1 as its input.

        the pipe symbol (|) is used to pass the output of one command as input to another command. This allows for the creation of powerful command chains and workflows by combining simple commands in a flexible and efficient manner.

    =>File Permissions keywords

        Read (r)       = 4       allows a file to read
        Write(w)       = 2       allows a file to modified
        Execute (x)    = 1       allows the execution of a file 

        ---------------------------------------------------------------------
        Meaning of r, w, and x
        r (Read):
            For files: Grants permission to read the file's contents.
            For directories: Grants permission to list the contents of the directory.
            Numeric value: 4
        w (Write):
            For files: Grants permission to modify the file's contents.
            For directories: Grants permission to add, delete, or rename files within the directory.
            Numeric value: 2
        x (Execute):
            For files: Grants permission to execute the file as a program.
            For directories: Grants permission to enter the directory and access its contents.
            Numeric value: 1

        The sudo command in Unix-like operating systems allows a permitted user to execute a command as the superuser (root) or another user, as specified by the security policy. The sudo command is commonly used to perform administrative tasks, which require higher privileges than those granted to a normal user.

        How sudo Works
        User Authentication:

        When you run a command with sudo, you may be prompted to enter your user password. This is to authenticate that you have the necessary permissions to execute the command with elevated privileges.
        Privileges:

        Once authenticated, sudo grants temporary elevated privileges to run the specified command as the superuser or another specified user.
        Security Policies:

        The /etc/sudoers file defines the rules and permissions for which users can run which commands via sudo. It specifies who can use sudo and what commands they are allowed to run.
